# Biz-helper

Telegram-ассистент для ремесленника. Бот приветствует пользователя, предлагает ознакомиться с работами мастера и при желании связаться с ним — отправляет уведомление мастеру в Telegram.

## Архитектура

Проект состоит из двух сервисов, взаимодействующих через gRPC:
┌─────────────────────────────────────────────────────────────────┐
│ Telegram API │
└─────────────────────────────────────────────────────────────────┘
│
▼
┌─────────────────────────────────────────────────────────────────┐
│ Bot Gateway (Сервер 1) │
│ ├── HTTP :8081 ← Telegram long polling │
│ ├── gRPC :50052 ← SendMessage от Logic Server │
│ └── gRPC клиент → :50051 (ProcessUpdate) │
└─────────────────────────────────────────────────────────────────┘
│
│ gRPC
▼
┌─────────────────────────────────────────────────────────────────┐
│ Logic Server (Сервер 2) │
│ ├── HTTP :8080 ← Админка (заглушка) │
│ ├── gRPC :50051 ← ProcessUpdate от Bot Gateway │
│ └── gRPC клиент → :50052 (SendMessage) │
│ └── PostgreSQL ← хранение данных │
└─────────────────────────────────────────────────────────────────┘

## Стек технологий

| Компонент                   | Технология                                |
| --------------------------- | ----------------------------------------- |
| Язык                        | Go 1.25                                   |
| Telegram API                | [telebot.v4](https://gopkg.in/telebot.v4) |
| Коммуникация                | gRPC                                      |
| База данных                 | PostgreSQL                                |
| Кэш (готов к использованию) | Redis                                     |
| Миграции                    | [goose](https://github.com/pressly/goose) |
| Контейнеризация             | Docker, docker-compose                    |
| Конфигурация                | `.env` + `.yml`                           |

## Конфигурация

### Переменные окружения (`.env`)

Пример содержимого `.env` в корне проекта:

```env
# Общие
ENV=development

# Bot Gateway
BOT_CONFIG_PATH=./bot/yml_configs/botConfig.yml
BOT_GRPC_SERVER_CONFIG_PATH=./bot/yml_configs/botGrpcServerConfig.yml
BOT_SERVER_CONFIG_PATH=./bot/yml_configs/botServerConfig.yml

# Logic Server
GRPC_CLIENT_CONFIG_PATH=./server/yml_configs/grpcClientConfig.yml
GRPC_SERVER_CONFIG_PATH=./server/yml_configs/grpcServerConfig.yml
SERVER_CONFIG_PATH=./server/yml_configs/serverConfig.yml

# PostgreSQL
POSTGRES_HOST=postgres
POSTGRES_PORT=5432
POSTGRES_USER=user
POSTGRES_PASSWORD=pass
POSTGRES_DB=db


YAML конфиги
Каждый сервис имеет свои .yml конфиги с настройками портов, ссылок и т.д.
Пути к ним задаются через .env.

-----------------------------------------
Запуск проекта
Требования:

Docker и Docker Compose
Go 1.25 (для локальной разработки)
-----------------------------------------
Запуск через Docker Compose
После запуска будут доступны:

Bot Gateway — принимает обновления от Telegram

Logic Server — обрабатывает бизнес-логику

PostgreSQL — база данных
-----------------------------------------
Миграции
Миграции находятся в server/migrations/. Для их выполнения используется goose.
-----------------------------------------
Сценарий работы
Пользователь отправляет команду /start

Бот приветствует пользователя и показывает инлайн-кнопку «Посмотреть работы»

При нажатии на кнопку открывается ссылка на Instagram/страницу мастера (задаётся в конфиге)

После возвращения в бот, пользователю предлагается выбор: «Связаться с мастером»

Если пользователь выбирает связь, бот отправляет мастеру уведомление в Telegram с данными пользователя (имя, username, ID)
-----------------------------------------

Текущий статус
Базовая архитектура (2 сервиса, gRPC)

Интеграция с Telegram API

Обработка /start и инлайн-кнопки

Отправка уведомления мастеру

PostgreSQL и миграции

Бизнес-логика (в разработке)

Логирование (планируется)

Unit-тесты (планируются)


```
